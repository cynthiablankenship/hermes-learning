# M01 - Linux & CLI Foundations

**Status:** 🔵 In Progress
**Prerequisites:** None — this is where we start.
**Time estimate:** 5-7 days at 1-2 hours per day

## Why This Module Matters

Almost every AI system runs on Linux. The DGX and HGX systems you work with at Dell run Linux. When you SSH into a customer's server to read logs, you're on Linux. Docker containers run on Linux. Python scripts run on Linux.

If you're comfortable on the Linux command line, everything else in this curriculum becomes 10x easier.

## What You'll Learn

By the end of this module, you will be able to:
- Navigate a Linux filesystem confidently
- Create, move, copy, and delete files and directories
- Read and search through files (including logs)
- Understand file permissions and ownership
- Monitor running processes
- Use pipes and redirection to chain commands
- SSH into remote servers

## Lessons

1. [[M01L01 - What is Linux]] — What Linux is, why it's everywhere in AI, and how it differs from Windows
2. [[M01L02 - Navigating the Filesystem]] — `pwd`, `ls`, `cd`, absolute vs relative paths
3. [[M01L03 - File Operations]] — `touch`, `mkdir`, `cp`, `mv`, `rm`, and doing it safely
4. [[M01L04 - Reading & Searching Files]] — `cat`, `less`, `head`, `tail`, `grep`, and why this matters for your job
5. [[M01L05 - Permissions & Ownership]] — `chmod`, `chown`, reading permission strings, why `permission denied` happens
6. [[M01L06 - Processes & Monitoring]] — `ps`, `top`, `htop`, `kill`, understanding what's running on a system
7. [[M01L07 - Pipes & Redirection]] — `|`, `>`, `>>`, `<`, and how to chain commands together
8. [[M01L08 - Environment Variables & PATH]] — What env vars are, how PATH works, `export`, `.bashrc`
9. [[M01L09 - SSH & Remote Access]] — Connecting to remote servers, keys vs passwords, basic SSH config
10. [[M01L10 - Lab - Navigating a DGX via SSH]] — Hands-on lab tying everything together
